# MeeLike Platform — System Overview for Payment Partners

> เอกสารนี้จัดทำเพื่ออธิบายภาพรวมของระบบ MeeLike ให้กับพันธมิตรด้านการเงิน (ธนาคาร / Payment Gateway)
> เน้นอธิบาย Business Model, กระแสเงิน (Money Flow), และความต้องการด้าน Payment ของระบบ

---

## 1. MeeLike คืออะไร

MeeLike เป็น **แพลตฟอร์มตัวกลาง** สำหรับ **บริหารจัดการทีมงาน Social Media Engagement** โดยเชื่อมระหว่าง **แม่ทีม (Team Leader)** กับ **ลูกทีม (Team Member)** พร้อมระบบการเงิน การจ่ายค่าจ้าง และการตรวจสอบงาน

### หลักการทำงาน

- **แม่ทีม** เป็นผู้ที่ **หาลูกค้ามาเอง** (จากช่องทางภายนอก เช่น LINE, Facebook, ปากต่อปาก ฯลฯ) จากนั้นนำงานเข้ามาบริหารจัดการผ่านระบบ MeeLike — กระจายงานให้ลูกทีม ตรวจงาน และจ่ายเงินให้ลูกทีม
- **ลูกค้าไม่ได้เข้ามาใช้งานเว็บ MeeLike โดยตรง** — ลูกค้าติดต่อและจ่ายเงินให้แม่ทีมโดยตรง
- **MeeLike ทำหน้าที่เป็นเครื่องมือจัดการงานและการเงินระหว่างแม่ทีมกับลูกทีม**

### บริการที่ให้

ให้บริการเพิ่ม Engagement บน Social Media ครอบคลุม 5 แพลตฟอร์ม โดยใช้คนจริง (ลูกทีม) ทำงานทั้งหมด:

| แพลตฟอร์ม | ประเภทบริการ |
|-----------|-------------|
| Facebook  | Like, Comment, Follow, Share, View |
| Instagram | Like, Comment, Follow, Share, View |
| TikTok    | Like, Comment, Follow, Share, View |
| YouTube   | Like, Comment, Follow, Share, View |
| Twitter   | Like, Comment, Follow, Share, View |

### สรุป: MeeLike คือเว็บอะไร?

> **MeeLike ไม่ใช่เว็บขายบริการให้ลูกค้าโดยตรง**
>
> MeeLike คือ **เว็บตัวกลางสำหรับบริหารจัดการทีม** — ให้แม่ทีมใช้จัดการงาน จ่ายค่าจ้าง ตรวจงาน และให้ลูกทีมใช้รับงาน ส่งงาน ถอนเงิน
>
> ลูกค้าอยู่ **ภายนอกระบบ** — แม่ทีมหาลูกค้ามาเอง รับเงินจากลูกค้าเอง แล้วมาบริหารงานผ่าน MeeLike

## 3. Business Model — แหล่งรายได้ของแพลตฟอร์ม

MeeLike มีรายได้ 3 ทาง:

### 3.1 Subscription Fee (ค่าสมาชิกรายเดือน — แม่ทีมจ่าย)

| แพลน | ราคา/เดือน | จำนวนทีม | Analytics | ฟีเจอร์เสริม |
|------|-----------|----------|-----------|-------------|
| Free | ฿0 | 2 ทีม | Basic | — |
| Basic | ฿49 | 5 ทีม | Basic | — |
| Pro | ฿99 | 20 ทีม | Pro | Export, Webhook, White Label |
| Business | ฿399 | ไม่จำกัด | Pro | ทุกอย่าง + Custom Domain + Priority Support |

### 3.2 Platform Fee (ค่าธรรมเนียมแพลตฟอร์ม — หักจากแม่ทีม)

คิดเป็น % จากยอดใช้จ่ายของแม่ทีม โดยอิงจาก Rank (คำนวณจากยอดจ้างเฉลี่ย 3 เดือนย้อนหลัง):

| Rank | ยอดจ้างเฉลี่ย/เดือน | ค่าธรรมเนียม |

| Bronze | ฿0 — ฿20,000 | **12%** |
| Silver | ฿20,000 — ฿50,000 | **11%** |
| Gold | ฿50,000 — ฿150,000 | **10%** |
| Platinum | > ฿150,000 | **9%** |

### 3.3 Withdrawal Fee (ค่าธรรมเนียมถอนเงิน — หักจากลูกทีม)

| รายการ | รายละเอียด |
|-------|-----------|
| **ค่าธรรมเนียมถอน** | **฿15 ต่อครั้ง** (คงที่) |
| **ยอดถอนขั้นต่ำ** | **฿100** |

> หมายเหตุ: อัตราค่าธรรมเนียมถอนปัจจุบันอยู่ระหว่างการเจรจากับ Payment Partner เพื่อให้ได้เรทที่ดีขึ้นในอนาคต

## 5. Order & Payment Lifecycle — วงจรการสั่งซื้อและชำระเงิน

### 5.1 ขั้นตอนจากรับงานลูกค้า ถึงลูกทีมได้รับเงิน

```
Step 1: แม่ทีมรับงานจากลูกค้า (เกิดนอกระบบ MeeLike)
   │  ลูกค้าติดต่อแม่ทีมโดยตรง (LINE, Facebook, โทร ฯลฯ)
   │  ตกลงบริการ ราคา จำนวน → ลูกค้าจ่ายเงินให้แม่ทีมโดยตรง
   ▼
Step 2: แม่ทีมสร้างออเดอร์ในระบบ MeeLike (เริ่มเข้าระบบ)
   │  แม่ทีม login เข้า MeeLike → สร้างออเดอร์ บันทึกรายละเอียดงาน
   │  ▸ Order status: pending → processing
   ▼
Step 3: แม่ทีมสร้าง Job มอบหมายให้ลูกทีม
   │  ▸ ระบบสร้าง Job → กระจายงานให้ลูกทีมในสังกัด
   ▼
Step 5: ลูกทีมรับงาน (Claim)
   │  ลูกทีมเห็น Job → กด Claim → ทำงาน (เช่น กดไลค์ คอมเมนต์)
   │  ▸ Job status: pending → in_progress
   ▼
Step 6: ลูกทีมส่งงาน (Submit)
   │  ลูกทีมทำเสร็จ → ส่งหลักฐาน (screenshot)
   │  ▸ Job status: in_progress → pending_review
   ▼
Step 7: แม่ทีมตรวจงาน (Review)
   │  แม่ทีมตรวจผลงาน → Approve หรือ Reject
   │  ▸ Approve → สร้าง Payout ให้ลูกทีม (pending)
   │  ▸ Reject → ลูกทีมต้องทำใหม่
   ▼
Step 8: แม่ทีมจ่ายเงินลูกทีม (Process Payout)
   │  แม่ทีมกด Process Payout → เงินหักจาก Wallet แม่ทีม → เข้า Balance ลูกทีม
   │  ▸ Payout status: pending → completed
   │  ▸ Order status: processing → completed (เมื่อทุก item เสร็จ)
   ▼
Step 9: ลูกทีมถอนเงิน (Withdraw)
   │  ลูกทีมถอนเงินจาก Balance → โอนไปบัญชีธนาคาร/PromptPay
   │  ▸ หักค่าธรรมเนียมถอน ฿15/ครั้ง (ขั้นต่ำถอน ฿100)
   ▼
Done ✓
```

### 5.2 Order Status Lifecycle

```
pending ──▶ confirmed ──▶ processing ──▶ completed
                                    └──▶ cancelled
```

### 5.3 Payment Status Lifecycle

```
pending ──▶ paid ──▶ refunded (กรณียกเลิก)
```

## 6. Wallet System — ระบบกระเป๋าเงิน

### 6.1 Wallet แม่ทีม

| รายการ | คำอธิบาย |
|-------|---------|
| **Balance** | คำนวณจากผลรวมของทุก Transaction (เงินเข้า − เงินออก) |
| **เงินเข้า** | Top-up, รายได้จากออเดอร์ |
| **เงินออก** | ค่าจ้างลูกทีม, ค่าธรรมเนียมแพลตฟอร์ม |

### 6.2 Balance ลูกทีม

| รายการ | คำอธิบาย |
|-------|---------|
| **Available Balance** | ยอดเงินที่ถอนได้ |
| **Pending Balance** | ยอดเงินที่รอแม่ทีมอนุมัติ/จ่าย |
| **Total Earned** | ยอดรายได้สะสมทั้งหมด |

## 7. ช่องทางชำระเงินที่ต้องการ (Payment Channels Required)

### 7.1 เงินเข้าระบบ (Collection)

| ช่องทาง | ใช้สำหรับ | รายละเอียด |
|--------|----------|-----------|
| **PromptPay QR** | แม่ทีม Top-up, ค่าสมาชิกรายเดือน | สแกน QR Code ชำระเงิน |
| **Bank Transfer** | แม่ทีม Top-up | โอนเงินผ่านธนาคาร |

### 7.2 เงินออกจากระบบ (Disbursement)

| ช่องทาง | ใช้สำหรับ | รายละเอียด |
|--------|----------|-----------|
| **PromptPay** | ลูกทีมถอนเงิน | โอนเงินไปเบอร์โทร/เลขบัตรประชาชน |
| **Bank Transfer** | ลูกทีมถอนเงิน | โอนเงินไปบัญชีธนาคาร |

### 7.3 ฟีเจอร์ที่ต้องการจาก Payment Gateway

| ฟีเจอร์ | ความสำคัญ | รายละเอียด |
|--------|----------|-----------|
| **QR Payment (PromptPay)** | สูง | รับชำระเงินผ่าน QR Code (ช่องทางหลัก) |
| **Payment Verification** | สูง | ยืนยันการชำระเงินอัตโนมัติ (แทนการตรวจสลิปด้วยคน) |
| **Disbursement / Payout** | สูง | จ่ายเงิน (ถอนเงิน) ให้ลูกทีมผ่าน PromptPay หรือ Bank Transfer |
| **Webhook Notification** | สูง | แจ้งเตือนเมื่อชำระเงินสำเร็จ / ถอนเงินสำเร็จ เพื่ออัปเดตสถานะอัตโนมัติ |
| **Refund API** | ปานกลาง | คืนเงินอัตโนมัติกรณียกเลิกออเดอร์ |
| **Settlement Report** | สูง | รายงานการเงินสำหรับ Reconciliation |

## 8. KYC & Compliance — การยืนยันตัวตน

ระบบมี KYC 3 ระดับ เพื่อจำกัดความเสี่ยงและปฏิบัติตามกฎหมาย:

| ระดับ KYC | ข้อมูลที่ต้องการ | สิทธิ์ที่ได้ |
|----------|----------------|------------|
| **Basic** (ยืนยันเบอร์โทร) | เบอร์โทร + OTP | แม่ทีม: เติมเงินได้ |
| **Verified** (ยืนยันตัวตน) | บัตรประชาชน + ภาพถ่ายคู่บัตร | ลูกทีม: ถอนเงินได้ (สูงสุด ฿10,000/วัน), สร้างทีมได้ |
| **Business** (ยืนยันธุรกิจ) | เอกสารจดทะเบียนธุรกิจ | ถอนเงินไม่จำกัด |

## 9. การจัดการเนื้อหาผิดกฎหมาย (Content Moderation & Risk Management)

> ธนาคาร/Payment Gateway มักถามว่า "ถ้ามีเพจผิดกฎหมายมาใช้บริการ จะจัดการอย่างไร?"
> หัวข้อนี้อธิบายกลไกป้องกันและจัดการทั้งหมดของ MeeLike

### 9.1 เนื้อหาที่ห้ามเด็ดขาด (Prohibited Content)

แพลตฟอร์ม **ปฏิเสธ** การให้บริการเพิ่ม Engagement กับเนื้อหาดังต่อไปนี้:

| หมวด | ตัวอย่าง |
|------|---------|
| **การพนันออนไลน์** | คาสิโน, แทงบอล, สล็อต, หวยออนไลน์, บาคาร่า |
| **เว็บไซต์/บริการผิดกฎหมาย** | ยาเสพติด, อาวุธ, สินค้าละเมิดลิขสิทธิ์ |
| **โฆษณาหลอกลวง / Scam** | แชร์ลูกโซ่, ลงทุนผลตอบแทนสูงเกินจริง, หลอกให้โอนเงิน |
| **เนื้อหาสำหรับผู้ใหญ่** | เนื้อหาลามกอนาจาร, สื่อไม่เหมาะสม |
| **อื่นๆ** | เนื้อหาที่ขัดต่อกฎหมายไทย หรือ นโยบายของ Social Media แต่ละแพลตฟอร์ม |

### 9.2 มาตรการป้องกัน (Prevention)

```
ก่อนเข้าใช้งาน
─────────────────────────────────────────────────────────────
1. KYC — แม่ทีมและลูกทีมต้องยืนยันตัวตน (เบอร์โทร, บัตรประชาชน)
         ทำให้ตรวจสอบย้อนกลับได้ว่าใครเป็นใคร
2. ข้อตกลง — แม่ทีมต้อง "ยอมรับข้อตกลง" (Guidelines Agreement)
              ก่อนสร้างทีมหรือเริ่มใช้งาน ระบุชัดเจนว่า
              ห้ามนำเนื้อหาผิดกฎหมายมาใช้บริการ
3. แจ้งบทลงโทษ — แสดงบทลงโทษชัดเจนตั้งแต่ตอนสมัคร
                  เพื่อให้ผู้ใช้ตระหนักถึงความเสี่ยง
```

### 9.3 กลไกตรวจจับและรายงาน (Detection & Reporting)

```
ใครรายงานได้?
─────────────────────────────────────────────────────────────
• ลูกทีม — เห็น Job ที่เนื้อหาผิดกฎหมาย → กดรายงานได้ทันที
• แม่ทีมอื่น — เห็นร้านที่มีเนื้อหาต้องสงสัย → รายงานได้
• Admin — ตรวจตราเชิงรุกจากระบบหลังบ้าน

ขั้นตอนการรายงาน:
─────────────────────────────────────────────────────────────
1. ผู้พบเห็น → กดปุ่ม "รายงานเนื้อหา" ในระบบ
2. เลือกหมวดหมู่ (การพนัน / ผิดกฎหมาย / Scam / ผู้ใหญ่ / อื่นๆ)
3. กรอกรายละเอียด + แนบหลักฐาน (ถ้ามี)
4. ระบบส่งรายงานไปให้ Admin ตรวจสอบ
```

### 9.4 ขั้นตอนตรวจสอบโดย Admin (Review Process)

```
                      ┌─────────────────┐
                      │  รายงานเข้ามา    │
                      │ (Report Pending) │
                      └────────┬────────┘
                               │
                      Admin ตรวจสอบหลักฐาน
                      ตรวจ URL, เนื้อหา, ข้อมูล
                               │
               ┌───────────────┼───────────────┐
               │               │               │
       ┌───────▼──────┐ ┌─────▼──────┐ ┌──────▼──────┐
       │   ยืนยัน      │ │  ยกเลิก    │ │  รายงานเท็จ  │
       │ (Confirmed)  │ │(Dismissed) │ │(False Report)│
       └───────┬──────┘ └────────────┘ └──────┬──────┘
               │                               │
       ดำเนินการลงโทษ                  ลงโทษผู้รายงานเท็จ
       (ดูหัวข้อ 9.5)                 (แบนการรายงาน 30 วัน)
```

**ระบบติดตามความน่าเชื่อถือของผู้รายงาน:**

| ระดับ | เงื่อนไข | ผลกระทบ |
|-------|---------|--------|
| **Trusted** | รายงานถูกต้อง ≥3 ครั้ง, ไม่เคยรายงานเท็จ | รายงานถูกจัดลำดับ priority สูง |
| **Normal** | ค่าเริ่มต้น | ตรวจสอบตามลำดับปกติ |
| **Suspicious** | เคยรายงานเท็จ ≥1 ครั้ง | ถูกตรวจสอบละเอียดขึ้น / อาจถูกแบนการรายงาน |

### 9.5 บทลงโทษ (Penalties)

เมื่อ Admin ยืนยันว่าเนื้อหาผิดกฎหมายจริง จะดำเนินการตามลำดับความรุนแรง:

| ลำดับ | การดำเนินการ | รายละเอียด |
|-------|-------------|-----------|
| **1** | **ระงับบัญชีทั้งทีมถาวร** | แม่ทีมและลูกทีมทั้งหมดในทีมนั้นถูกระงับ ไม่สามารถสร้างออเดอร์หรือรับงานได้อีก |
| **2** | **ริบเงินค้างถอนทั้งหมด** | ยอดเงินในระบบ (ทั้ง Wallet แม่ทีม และ Balance ลูกทีม) ถูกอายัด |
| **3** | **ดำเนินคดีตามกฎหมาย** | กรณีร้ายแรง — ส่งข้อมูล KYC และหลักฐานให้เจ้าหน้าที่ดำเนินคดี |

### 9.6 ผลกระทบทางการเงินเมื่อตรวจพบ (Financial Impact)

```
เมื่อ Admin ยืนยันเนื้อหาผิดกฎหมาย:
─────────────────────────────────────────────────────────────
1. ระงับบัญชีแม่ทีม → สถานะ "suspended" → สร้างออเดอร์ใหม่ไม่ได้
2. ระงับบัญชีแม่ทีม → เติมเงิน/ถอนเงินไม่ได้
3. ระงับบัญชีลูกทีมทั้งทีม → ถอนเงินไม่ได้
4. อายัดเงินในระบบ → รอการตรวจสอบ/ดำเนินคดี
5. ออเดอร์ที่ค้าง → ยกเลิกทั้งหมด
6. แจ้ง Payment Gateway (ถ้ามี) → ระงับธุรกรรมที่เกี่ยวข้อง
```

### 9.7 สรุปภาพรวม — ทำไมแพลตฟอร์มจึงมีความเสี่ยงต่ำ

| มาตรการ | คำอธิบาย |
|---------|---------|
| **KYC บังคับ** | ผู้ใช้ทุกคนต้องยืนยันตัวตน (เบอร์โทร/บัตรประชาชน) จึงตรวจสอบย้อนกลับได้ |
| **ยอมรับข้อตกลงก่อนใช้งาน** | แม่ทีมต้องอ่านและยอมรับกฎเนื้อหาต้องห้ามก่อนเริ่มใช้งาน |
| **ระบบรายงานหลายชั้น** | ลูกทีมเห็นเนื้อหาผิดกฎหมายสามารถรายงานได้ทันที + Admin ตรวจตราเชิงรุก |
| **ลงโทษรุนแรงและรวดเร็ว** | ระงับถาวร + ริบเงิน + ดำเนินคดี ไม่มีแค่ "เตือน" |
| **เงินไม่ออกจากระบบทันที** | ลูกทีมต้องรอ Payout จากแม่ทีม + มีค่าธรรมเนียมถอน ทำให้มีจุดตรวจสอบหลายจุดก่อนเงินออก |
| **KYC gate ถอนเงิน** | ต้องยืนยันบัตรประชาชนก่อนถอนเงินได้ — ผู้กระทำผิดต้องเปิดเผยตัวตนจริง |

## 10. ตัวอย่างกระแสเงินจริง (Example Transaction Flow)

### Scenario: แม่ทีมรับงาน Instagram Likes 1,000 ตัว จากลูกค้า

```
ราคาที่ตกลงกับลูกค้า: ฿500 (฿0.50/like)
ค่าจ้างลูกทีม: ฿300 (฿0.30/like)

1. ลูกค้าจ่าย ฿500 ให้แม่ทีมโดยตรง ──────────────── (นอกระบบ MeeLike)
2. แม่ทีมเติมเงิน ฿500 เข้า Wallet ───────────────── (Top-up)
3. แม่ทีมสร้าง Job → ลูกทีม 5 คน คนละ 200 likes
4. ลูกทีมทำงานเสร็จ → แม่ทีม Approve
5. แม่ทีมจ่ายลูกทีม:
   ├── ลูกทีมรวม: ฿300 ─────────────────────────────── (เงินออกจาก Wallet แม่ทีม)
   └── Platform Fee 12% (Bronze): ฿300 × 12% = ฿36 ─── (รายได้ MeeLike)
6. แม่ทีมกำไร: ฿500 - ฿300 - ฿36 = ฿164

7. ลูกทีมถอนเงิน (สมมุติ ลูกทีม 1 คน ถอน ฿100):
   ├── ยอดถอน: ฿100 (ขั้นต่ำ ฿100)
   ├── Withdrawal Fee: ฿15 (คงที่ต่อครั้ง) ────────────── (รายได้ MeeLike)
   └── ลูกทีมได้รับจริง: ฿85

═══════════════════════════════════════════════════════════
สรุปรายได้ MeeLike จาก Transaction นี้:
   • Platform Fee: ฿36
   • Withdrawal Fee: ฿15 (×5 ลูกทีม = ฿75)
   • รวม: ฿111 จากออเดอร์ ฿500 (~22.2%)
═══════════════════════════════════════════════════════════
```

## 12. สิ่งที่ต้องการจาก Payment Partner

### ความต้องการหลัก

1. **QR Payment Collection** — รับชำระเงินจากแม่ทีม (Top-up, ค่าสมาชิก) ผ่าน PromptPay QR พร้อม Webhook แจ้งสถานะอัตโนมัติ (ไม่มีระบบบัตรเครดิต)
2. **Disbursement / Payout** — ระบบถอนเงินให้ลูกทีม ผ่าน PromptPay หรือ Bank Transfer (รายคนหรือหลายคนพร้อมกัน)
3. **Refund** — คืนเงินอัตโนมัติกรณียกเลิกออเดอร์
4. **Reporting & Reconciliation** — รายงานการเงินสำหรับตรวจสอบความถูกต้อง

### Integration ที่ต้องการ

- REST API สำหรับสร้าง PromptPay QR Code และตรวจสอบสถานะ
- REST API สำหรับถอนเงิน (Disbursement) ผ่าน PromptPay / Bank Transfer
- Webhook callback เมื่อชำระเงินสำเร็จ / ถอนเงินสำเร็จ
- Dashboard สำหรับดูสถานะธุรกรรม
- Sandbox environment สำหรับทดสอบ
